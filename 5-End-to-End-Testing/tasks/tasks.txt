Итоговое задание по курсу:
1. Запустить локально и изучить приложение RealWorld Example App из папки frontend.
   Проект реализован на "чистом" JS + Web Components.

2. Выбрать любой класс в приложении RealWorld Example App и покрыть его модульными тестами.
   У выбранного класса требуется тестировать контракт.
   Для определения контракта можно использовать определение тестовых сценариев для каждого метода класса по трем потокам:
     - Normal flow
     - Error flow
     - Exception flow
   Минимально в классе должно быть покрыто модульными тестами 2-3 метода.

3. Для выбранного в п.2 класса требуется написать интеграционные тесты (используемые в классе зависимости не изолировать).
   Для каждого тестируемого метода требуется определить тестовые сценарии по трем потокам и написать соответствующи тесты: 
     - Normal flow
     - Error flow
     - Exception flow
   Минимально в классе должно быть покрыто интеграционными тестами 2-3 метода.
   Методы могут быть как те же, которые были выбраны в п.2 для покрытия модульными тестами, так и другие.
   Опционально (не обязательно):
     - Создать серверный билд с использованием GitHub Actions, настроить CI для запуска тестов при публикации кода в репозиторий.
     - Для изоляции тестового окружения (бэкенда для приложения) воспользоваться Docker-образами, доступными в папке backend, в которой находится исходный код серверной части приложения RealWorld Example App.
